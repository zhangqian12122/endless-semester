# 无限学期：浏览器灰盒原型 V1.8

> 在线试玩：<https://zhangqian12122.github.io/endless-semester/>

这是一个可直接运行的卡牌构筑游戏原型。主题是“学生 × 校园梗 × 宠物羁绊”，目标不是复刻现有作品的美术或数值，而是先验证这些核心体验：

- 卡牌一句话看懂：3 能量、每回合抽 5 张、敌人意图完全公开。
- 顶栏资源使用统一的小型校园图形：校园币是带书页纹样的校徽硬币，卡组与物品分别使用叠牌和书包图标；中文数值始终保留，不依赖 emoji 识别。
- 战斗意图会实时计算“敌方总攻击 - 当前护甲 + 无视护甲损失”，直接显示预计掉血、结束后生命与致命预警；预览按真实结算顺序封顶生命损失，情绪内耗先致死时不会再虚算敌方攻击或护甲抵挡，且全程只读。
- 玩家生命降到 0 时始终判定战败；即使同一张自伤牌也把敌人击倒，也不能以 0 生命进入战后奖励。
- 学生初始卡组有 4 攻、4 防和 2 张职业特性牌。
- 第 1 学期第 1 周的教学战会把攻击、防御与当前主角特性牌固定在首手前三张；只改变这一次展示顺序，后续战斗、挑战战和剩余牌序仍正常随机。
- 宕机鸭通过战斗充能，每场可用“重启猛啄”主动出手一次；技能入口收敛为能量上方的鸭鸭图标，直接显示当前充能进度，点击可查看效果并在满足条件时发动，胜利继续累积长期羁绊。
- 三种星座学生开局：白羊首击爆发、双子零费循环、巨蟹稳定防守。
- 当前正式主角为“嘉豪”：白羊、双子、巨蟹三种开局共用 `student-jiahao-v1.webp` 外观，但星座被动、专属卡池与构筑玩法仍各自独立；宠物“宕机鸭”分别使用完整战场图与充能头像；六名普通敌人“瞌睡虫 / 作业团 / 闹钟怪 / 手机精 / 群聊99+ / 卡纸打印机”各自使用独立战斗立绘。图片加载失败时仍会显示 CSS 灰盒轮廓，不会阻断战斗。
- 每学期开始从战车、力量、隐者三张塔罗命运牌中选择一张；每张牌都同时公开收益与代价，本学期内不能更换。
- 战车提供首回合 1 点额外能量但让敌人生命 +15%；力量让宠物初始充能 +1 但敌人每段攻击 +2；隐者首回合多抽 2 张但敌人初始获得 8 护甲。
- 塔罗契约会持续显示在地图、战斗和规则页；战斗中与当前星座被动合并为一组紧凑图标，悬停或聚焦再展开完整收益与代价，进入下一学期后重新选择，V1.6 旧存档首次载入时可在当前周补选一次。
- 每张塔罗在休息节点拥有一次“本学期共鸣”：战车以 6 生命换一次删牌，力量以 30 币换羁绊 +4 与恢复 6，隐者以 35 币换一次升级与恢复 8。
- 塔罗共鸣会消耗本次休息机会，且每学期只能成功使用一次；资源不足、卡组已到下限或没有可升级牌时会明确禁用并说明原因。
- 点击共鸣后会先展示生命、金币、羁绊、卡组或升级数量的真实前后变化；确认前可无损取消，不会因误触提前扣除资源。
- 每个星座首批 4 张专属牌；普通战后固定出现 1 张本星座专属和 2 张普池牌。
- 商店固定进货 1 张本星座专属和 2 张普池牌，绝不会出现其他星座的专属牌。
- 第 8 周精英战后触发一次星座刻印：白羊攻击每段 +2、双子费用 -1、巨蟹基础护甲 +3。
- 刻印只提供给本场实际使用过的合格卡牌；每张牌最多一个刻印，但可与普通升级同时存在。
- 宠物羁绊不再自动堆伤害：羁绊 3 时从奶凶、护崽、叼笔记三条路线中选择，羁绊 10 与 25 时强化。
- 三条宠物路线分别提供受控的额外伤害、护甲或抽牌，仍然每场最多出手一次并消耗 1 能量。
- 战斗中可随时查看抽牌堆与弃牌堆；两者分别固定在手牌左右并使用不同的叠牌轮廓与数量标记，抽牌堆按牌名整理展示、不泄露随机牌序；打开后卡面保持正常亮度与完整规则文字，消耗堆不再作为常驻控件占用战斗空间。
- 抽牌堆耗尽且弃牌堆有牌时会按真实规则洗回，并显示“弃牌洗回抽牌堆 N 张”以及右侧弃牌堆飞向左侧抽牌堆的卡背动画；卡牌抽牌、宠物抽牌与新回合抽牌共用同一真实结算结果，减少动态效果时只保留文字。若两堆与手牌同时为空，“抽后弃牌”不会生成无法完成的弃牌要求，战斗仍可继续。
- 新增本地战绩页：记录胜率、平均回合、战斗掉血、出牌次数、卡牌选择及常用卡。
- 试玩统计只写入当前浏览器存档，不接入第三方埋点或追踪服务。
- 战斗结束后展示回合、出牌、实际伤害、掉血和针对当前敌人的复盘建议。
- 校园档案记录 8 种敌人的行动规律；首次遭遇后才公开图鉴内容。
- 11 项生涯成就跨对局保留，但不提供永久数值加成，避免滚雪球压制新玩家。
- 构筑诊断会把当前卡组识别为爆发强攻、稳固防守、高速循环或伙伴协同，并给出明确短板。
- 战后奖励与商店会为候选牌标注核心契合、补足短板、转型组件或中性选择，并解释原因。
- 选牌标签只读取当前构筑，不修改卡牌效果、奖励概率、随机数状态或商店价格。
- 每学期开始生成一张可预览、可存档的随机路线；同一存档不会通过刷新页面重随路线。
- 第 1、2、8、16 周保持固定教学与考试，其余周提供两个不同类型的节点，并保底至少两次休息和两次商店机会。
- 第 1 学期的新普通路线按周解锁敌人：第 3–5 周只随机“瞌睡虫 / 闹钟怪”；第 6–10 周加入会施加状态或塞牌的“作业团 / 手机精”；第 11–15 周再开放“群聊99+ / 卡纸打印机”，此时六种普通敌人全部可出现。
- 新生成路线会记住最近一次可选战斗的敌人；下一场普通战或挑战战会优先从其余敌人中抽取，避免连续几周重复同一套行动。敌人池只剩一种时仍会安全回退，不改变固定教学与考试。
- 这套解锁只约束新生成路线里的普通战。第 1、2 周教学战、第 8 周精英、第 16 周 Boss、固定事件节点、可绕开的挑战战与“校园怪谈”调查战都保留原来源；第 2 学期起普通路线也从第一周使用完整普通敌人池。
- 已保存的学期路线是存档事实：读取旧存档时不会按新周段替换敌人，也不会额外消耗随机数；只有缺失路线或进入下一学期时才按当前规则生成新路线。
- 地图默认把 16 周学期日历收纳为一条“当前周”入口，避免全览长期挤占路线高度；点击后以可关闭弹层展示桌面端 4×4、移动端 2 列的完整日历，已完成周画圈盖勾、当前周高亮，战斗、精英、挑战、事件、休息、商店与期末继续使用固定符号预告，并保留下一挑战信息。弹层支持关闭按钮、点击遮罩、`Esc`、`Tab` 焦点圈及关闭后焦点返回入口。
- 问号事件会禁用满血恢复、重复优惠、资源不足或无牌可选的空收益选项，并显示真实恢复量与禁用原因。
- 会立即扣除生命或校园币的确定性事件会先展示真实前后值；确认前可以无损取消，避免移动端误触直接支付代价。
- 普通物品收齐后，纸箱和储物柜不会再让玩家冒险换取空奖励；精英、怪谈与 Boss 的物品池耗尽时分别改为 50、70、100 校园币。
- 每学期期中前后各生成一次可绕开的挑战战：敌人生命 +35%、攻击伤害 +25%，数值在进入前完全公开。
- 挑战胜利不再固定发牌，而是从三条公开路线中选一条：“命盘补课”获得 20 币与本星座专属牌三选一，“伙伴特训”获得 25 币并在符合来源时提供宠物蛋、否则羁绊 +2；当前宠物达到 Lv.3 精通后不再堆无效羁绊，改为共 45 币。“失物招领”获得 10 币与未拥有随身物品。
- 随身物品路线通常提供未拥有普通物品二选一；只有少数指定挑战怪会把这条既有路线替换成一件确定招牌物品，不增加第四条路线或额外掉落。首个样板为“挑战手机精 → 静音手机”；已拥有后恢复普通二选一，全部普通物品也收齐后自动改为 45 校园币。
- 十一件随身物品会在物品卡上直接标明“每回合、每场、入场或每学期”的触发频率；静音手机每场第一回合多抽 1 张，且不进入商店、怪谈、精英或普通随机物品池。学生证统一降低卡牌、物品与删牌服务 10% 价格，创可贴在正好半血时也会触发，耳塞明确只抵挡每场第一次走神。
- 挑战奖励页新增“构筑参谋”：根据卡组厚度、当前主流派、宠物下一里程碑和书包空位标记一条本局推荐，并逐条解释判断。
- 推荐只读取当前状态，不修改奖励、概率或随机数；三条路线都会预览选择后的金币、羁绊或后续奖励，玩家仍可自由选择。
- 挑战胜利复盘会立即保存羁绊、试炼金币和奖励路线；路线页、专属牌候选或物品候选刷新后都从当前阶段继续，宠物里程碑与 45 币替代奖励不会重复结算。
- 每种星座在挑战战中拥有独立的可选试炼：白羊需在 3 回合内获胜，双子需单回合打出 4 张牌，巨蟹需把本场生命损失控制在 5 以内。
- 星座试炼会在地图、战斗进度和战后复盘中完整公开；完成额外获得 10 校园币，失败不影响挑战胜利和三条基础奖励路线。
- 校园档案新增星座试炼册：白羊、双子、巨蟹分别记录跨对局完成次数，首次完成会盖上对应印章。
- 三种印章全部收集后解锁「三星连珠」成就；试炼册与成就均不提供永久战力，只保留收藏和挑战目标。
- 挑战从三种词缀中抽取两种且同学期不重复：“限时下课”惩罚拖延，“桌面爆满”污染抽牌，“第一节早课”压缩首回合能量。
- 词缀名称与一句话效果会同时显示在路线预告、当前节点、战斗顶部和战后复盘；V1.1 旧路线自动补齐词缀且不重排节点。
- 随机战斗池新增“群聊99+”与“卡纸打印机”，分别强调多段伤害、状态污染和爆发时机。
- 六种普通敌人各自拥有不重复的短机制身份与公开说明：瞌睡虫强调攻防节拍、作业团污染待办、闹钟怪公开倒计时、手机精分心护屏、群聊99+把战斗区内的紧张转成最多额外 2 段轰炸、卡纸打印机把剩余护甲转成最多 6 点重击伤害。玩家清理紧张或击破打印机护甲后，意图数字会立即降低；能真正跨过蓄压断点的手牌会直接标出“重击 12→9 · 脱险 / 无伤 / 仍致命 / 预计掉血”，未越过 6 甲上限的破甲绝不虚报收益。打印机意图详情仍会标出距离下一次降伤还需击破多少护甲，预告与真实结算共用同一计算结果。手机精发动带走神的推送行动时会明确获得 4 或 3 点护甲，即使耳塞抵挡走神也照常护屏。
- 一学期 16 周，包含战斗、精英、问号事件、休息、商店和期末 Boss。
- 卡牌奖励、卡牌移除、卡牌升级、状态牌、随身物品与书包容量均已接入。
- 通过期末后可带着构筑进入下一学期，敌人和玩家按公开规则成长。
- 首场战斗包含三步教学，并提供六条核心规则页；回合流程说明收进新生教学，战场不再常驻单独的“第 N 回合”标签。教学遮罩是带标题、说明与 `Tab` 焦点圈的真正模态弹层，每步默认聚焦“下一条”，完成或跳过后把焦点交给敌人意图，同时立即保存已读状态，刷新不会重复打断。
- 每周开始自动保存于浏览器，可从标题页继续游戏。
- 新开局与进入下一学期时，强制塔罗选择页也会立即保存；关闭页面不会恢复旧周目或要求重打期末 Boss。
- 通过期末并完成总结后会保存“等待下一学期”状态；刷新或返回标题仍会回到期末总结，不会要求重打 Boss。
- 进入普通、挑战、精英、Boss 或怪谈战时会先保存开局检查点；战斗中刷新会从同一敌人、同一词缀和同一首手牌重新开战，不会退回地图改选路线。检查点只保证确定性重开，不保存回合中途进度。
- 进入商店会固定三张卡牌、两件物品与删牌价格；买牌、买物品和删牌服务逐笔保存，刷新后保持同一库存、售罄状态、金币和构筑，离店后才推进周数。
- 进入休息节点会立即锁定本周选择；普通升级、宠物里程碑和塔罗共鸣支付后的选牌阶段都会保存，刷新后从同一阶段继续，不能退回地图改选路线或重复领取收益。
- 奖励物品遇到满书包时，会保存已经选中的新物品和奖励来源；刷新后仍回到同一整理页面，可明确取消或替换一件旧物品，替换统计只结算一次。
- Boss 胜利出现复盘时就会建立期末结算存档；纪念物、宠物里程碑与主力牌升级会分阶段恢复，刷新不会重复发奖或跳过选择。
- 每张卡组副本独立累计“主力进度”：每场胜利中实际打出过的卡各获得 1 点，同一副本每场最多记录一次；本学期达到 `3/3` 的未升级牌可参加期末免费升级，进入下一学期后重新累计。构筑页与战后复盘会同步显示当前进度和还差几场，战斗卡牌类型栏用三枚小点区分同名副本的进度。
- 普通战胜利复盘会立即固定 15 币与三张候选牌；刷新后回到同一组战斗奖励，橡皮擦重抽后的新候选也会同步保存。
- 问号节点一旦揭示事件就会保存事件身份；在选择页或代价确认页刷新，会继续同一事件，上一事件记录也会跨刷新保留以维持防连续重复规则。
- 问号事件在扣除生命、校园币或附加下场战斗代价后，会固定升级、删牌、卡牌与物品候选；刷新后从同一选择页继续，不会重复支付代价或重抽奖励。
- 精英胜利复盘会立即固定 30 币、三张卡牌候选和本场实际使用过的刻印候选；卡牌、物品、刻印三个阶段分别保存，刷新只会从当前阶段继续，物品池耗尽的 50 币补偿也只结算一次。
- 怪谈调查战胜利复盘会立即固定物品候选；优先提供未拥有稀有物品，没有稀有时回退到一件其他物品，全部收齐后的 70 币补偿只结算一次。
- 已有对局存档时，开始新游戏必须先经过覆盖确认页；取消不会改写存档，生涯档案与成就也不会随新对局清空。
- 敌人意图压缩为头顶图标、数值与行动名，并去掉行动名外层的黑色大底；攻击、护甲和塞牌等复合行动会并排显示多个意图章。悬停、键盘聚焦或点击可展开真实的逐段伤害、护甲、走神、状态牌去向、怪物特性、应对建议和完整行动周期，点击后可固定，`Esc` 可关闭；应对建议会按当前行动、真实伤害倍率与机制进度变化，并结合当前手牌、能量和已充能宠物，只给本回合可直接执行的路线；建议只在玩家可操作时出现，结算阶段不会误导玩家。
- 双方当前护甲以生命条左侧的蓝色盾牌显示，零护甲时保留低对比占位；角色名字只负责身份识别，不再混入战斗数值。
- 701px 起的平板与桌面战斗把能量、宠物充能图标、结束回合和扇形手牌固定在首屏底部，抽牌堆与弃牌堆分列手牌左右，标准窗口无需下滑即可连续出牌；701–980px 会收起次要顶栏信息但保留生命、规则与构筑入口，战场宠物缩小并贴近学生脚边，不再抢主角权重，消耗堆也不常驻显示。

## 卡池规则

- 普池：所有星座均可获得，是攻击、防御、抽牌与宠物构筑的公共骨架。
- 专属池：只属于当前星座，用于强化该星座的核心节奏。
- 战后三选一：固定为 `1 专属 + 2 普池`，位置随机，不出售其他星座专属牌。
- 挑战“命盘补课”：固定从当前星座专属池抽取 3 张，不混入普池，也不跨星座。
- 指定类型事件：从当前星座专属池与普池混合抽取，仍然禁止跨星座。

## 宠物蛋与孵化（首个纵切片）

- 只有少数具有生物属性的可驯化怪物拥有对应的蛋与动物幼崽；手机精、闹钟怪和打印机等非生物敌人不会为了扩宠物数量强行掉蛋。招牌随身物品也只配置给少数挑战、精英或 Boss，不要求每只怪都掉东西。
- 击败可驯化怪物后，蛋可以作为独立的战后奖励路线出现，来源与幼崽种类在领取前公开；首个验证链为“挑战瞌睡虫 → 困困虫蛋 → 困困虫幼崽”。
- 不采用每只怪物都进行低概率暗抽的掉落方式。普通敌人负责积累“发现进度”，精英、挑战战或指定怪谈提供确定的蛋奖励，玩家能主动追踪目标。
- 宠物蛋进入独立孵化位，不占书包；携带完成 3 场有效战斗后孵化，失败不倒扣进度，休息节点可以查看蛋壳变化和剩余场次。
- 幼崽继承原怪物最有辨识度的一项机制，但改写为受控的玩家收益。例如困困虫幼崽偏蜷缩防御与慢速充能，不直接复制敌人的完整数值。
- 孵化只解锁宠物种类，不会在当前局强制替换当前伙伴；新一局在开始界面选择一只已解锁宠物，确认开局后整局锁定，羁绊与充能随新局重新培养。
- 已解锁对应幼崽或孵化位被占用时，挑战的伙伴奖励优先回退为当前参战伙伴羁绊 +2；伙伴达到 Lv.3 精通后再改为共 45 校园币，不继续堆积没有效果的羁绊。不会生成无效重复宠物，也不提供永久跨对局战力抽奖。

## 主要文件

- `game-data.js`：卡牌、敌人、物品和事件数据。
- `game-engine.js`：独立于界面的战斗、随机学期路线与长期成长规则。
- `career.js`：跨对局敌人发现、成就进度与生涯档案规则。
- `build-analysis.js`：构筑流派评分、短板判断与可执行建议。
- `app.js`：页面流程、奖励、事件、商店和交互。
- `styles.css`：响应式灰盒视觉。

### 敌人战斗立绘

- 正式敌人素材保存在 `assets/characters/`，使用带透明通道的 WebP；单张限制在 200KB 内，加载失败时保留原有 CSS 灰盒。
- 后续新增或重绘的怪物统一采用原创的二次元半厚涂方向：保留清楚剪影、夸张动作和机制道具，不做幼态 Q 版，也不复刻其他游戏的具体角色、构图或纹样；每只怪物先用一张完整样板确认审美，再批量扩展同组素材。
- `enemy-groupChat-v1.webp` 以三段向左刷出的通知流表现多段攻击，主体右侧保留失控消息堆，在小尺寸下仍能读出“连续刷屏”。
- `enemy-printerJam-v1.webp` 以向左推出的机械出纸重击表现蓄压爆发，同时保留右侧警示眼与中央卡纸结构；桌面与移动端分别使用独立缩放规则避免关键轮廓被裁切。
- `enemy-rivalShadow-v1.webp` 用书本、笔迹与拖影构成持续加速的学术幻影，动作明确朝向玩家，呼应每回合伤害递增的精英规则。
- `enemy-finalExam-v1.webp` 把试卷、时钟与红色批改结构组合成四阶段轮转的试卷堡垒，让 Boss 的公开循环在轮廓上也有记忆点。

## 当前边界

这是用于验证玩法的单机原型：三十张卡牌已完成首轮正式卡面，正式主角“嘉豪”、宠物“宕机鸭”、八名敌人与首张教室战场背景已经接入；其他场景、音效与角色动作仍处于灰盒或纵切片阶段。存档仅使用浏览器本地数据，也尚未接入远程数据埋点和数值编辑器。正式开发时建议先用真实试玩数据校准胜率、平均战斗回合和卡牌选择率，再扩充角色与宠物；不要先堆内容。
